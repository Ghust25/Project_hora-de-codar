Aula 10

Eventos

- Podemos ativar eventos nos componentes para dispararalgum método; - Um evento clássico que utilizamos muito é o click; - A sintaxe é: (click)="algumaFunção()" - Os métodos ficam na classe; - Futuramente utilizaremos este recurso para acessar uma API;

Aula 11

Emitindo eventos

- Podemos comunicar eventos de um componente filho para um componente pai; - Para isso vamos precisar do @Output, que é a saída do evento do componente filho; @Input do pai para o filho @Output do filho para o pai - Na tag de invocação do componente, escolhemos um método para executar quando o evento for emitido; - Exemplo: (emit)="onEmit()"

Aula 12

Renderização de listas

- Outro recurso importante é o loop em listas; - Para isso vamos precisar de uma propriedade com os itens da lista; - E no template do componente vamos utilizar o *ngFor; - A sintaxe é: *ngFor="let item of items" - Desta maneira podemos resnderizar template baseado em dados;

Aula 13

Importância das interfaces

- Toda entidade que vamos trabalhar precisa de uma interface; - Isso vai tornar nosso código mais simples ao longo do programa; - Padronizando ele e facilitando a manutenção; - Vamos implementar uma interface a nossa lista de animais; - E fazer um evento que também utilizará este recurso;

Aula 14

Pipe operators

- Os pipes operatos são recursos para trabalhar com strings nos templates; - POdemos realizar diversas funções, como deixar o texto em caixa alta; - A sintaxe é: {{dado | algumPipeOperator}} - Destta maneira nos poupamos de regras de CSS ou manipulação de strings com JS;

Aula 15

Two way data binding no Angular

- O Two way data binding é um recurso muito interessante para trabalho com formulários; - Basicamente conseguimos alterar props e o templare com o preenchimento de inputs; - É necessário importar o FormsModule no componente principal; - E também declarar o ngModel no input, além de preencher também o atributo name, todos com o mesmo valor;

Aula 16

Services do Angular

- O sevice é outra parte fundamental da arquitetura do Angular; - Geralmente ficam as requisições para as APIS que utilizamos no projeto; - Precisamos criar o service com: ng generate service - Importamos no componente e iniciamos no contructor; - Depois é possivel acessar os mètodos dele;

Aula 17

Angular Router

- Nós vamos criar um nivo arquivo para declarar nossas rotas; - Nele vamos importar os módulos Routermodule e Routes; - Após a definição estas rotas precisam ser importados em app.module.ts; - E por fim no template principal criamos as rotas; - E substituímos os componentes por:

Aula 18

Requisições HTTP

- Vamos realizar requisições HTTP nos nossos services; - Precisamos importar dois pacotes: HttpClient e HttpHeaders; - Neste exemplo utilizamos uma API local para simular as request, com o json-server; - Por último, é necessário inicializar o módulo HttpClientModule em app.module.ts;

Rodar o server: npm run server

Aula 19

Dynamic routes - selecionando dado individual

- Para resgatar um dado do banco vamos precisar criar uma rota dinâmica no nosso router; - Baseado no id do item teremos a seleção de dado no banco, geralmente o id vem pela URL; - Para resgatar parâmetro da URL vamos utilizar o ActivatedRoute, um módulo de Angula; - A lógica para requisição HTTP ficará no service;

Aula 20

Excluindo dados pelo Service

- Agora vamos refatorar nossa ação de remover serviço com a execução HTTP; - Criaremos outro método no service; - Eajustaremos a meneira que o método da classe do componente é executado; - Com simples mudanças estaremos integrando com a API!
